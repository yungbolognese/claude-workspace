## Example 6: Debug Logging Pattern

### Your Code Pattern
You frequently add debug logging:

```swift
// File 1
print("DEBUG: User data: \(userData)")

// File 2
print("DEBUG: API response: \(response)")

// File 3
print("DEBUG: View lifecycle: viewDidAppear")
```

### Detection

**Proposal: Debug Logger Utility**
- **Type**: Code Pattern
- **Occurrences**: 12 instances
- **Suggestion**: Create unified debug logger

### Generated Template
```swift
// DebugLogger.swift
struct DebugLogger {
    static func log(_ message: String, file: String = #file, line: Int = #line) {
        #if DEBUG
        let filename = (file as NSString).lastPathComponent
        print("[\(filename):\(line)] \(message)")
        #endif
    }
}

// Usage:
DebugLogger.log("User data: \(userData)")
```

**Benefits**:
- Consistent logging format
- Easy to disable in production
- File and line number tracking
- Single place to add features (timestamps, log levels, etc.)

## How to Use These Examples

### 1. Let Patterns Emerge Naturally
Don't force patterns. Just work normally and the meta-agent will detect repetition.

### 2. Review Proposals Critically
Not every pattern needs automation. Consider:
- How often will you use it?
- Is the time savings worth it?
- Will it reduce errors?
- Does it improve consistency?

### 3. Approve Selectively
```bash
# Review all proposals
python3 meta_agent.py proposals

# See detailed impact
python3 meta_agent.py report

# Approve the valuable ones
python3 approve_proposal.py proposal_cmd_1234
```

### 4. Iterate and Improve
After using a generated skill:
- Does it work well?
- Could it be improved?
- Should it handle edge cases?
- Should it be combined with other skills?

Tell Claude to enhance it!
